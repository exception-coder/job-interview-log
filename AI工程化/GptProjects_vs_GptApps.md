# GPT Project vs GPT Apps（极简核心对比表）

| 对比项 | **GPT Project** | **GPT Apps** |
|--------|------------------|--------------|
| 本质定位 | 工程级多文件项目 | 即用型 Workflow 应用 |
| 文件结构 | 多文件目录，可管理代码/资源 | 单流程/少文件 |
| 是否可写代码 | ✅ 可写完整代码、脚本 | ❌ 主要通过可视化流程 |
| 执行方式 | 运行脚本/程序，工程流水线 | 点击运行任务流 |
| 复杂度 | 高（适合系统级开发） | 低（适合工具类应用） |
| 扩展性 | 高，可接 API/库/外部系统 | 中，流程可叠加但有限 |
| 持久化 | 支持工程级持久化（代码+数据） | 存流程逻辑，不存复杂工程 |
| 适用人群 | 开发者 | 普通用户 |
| 典型用途 | Agent 系统、自动化平台、后端服务、多模块任务 | 日报工具、翻译器、文件处理、简单机器人 |
| 最终定位 | **构建“系统”** | **构建“工具”** |
