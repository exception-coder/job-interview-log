## 数据流对比

### 重构前：全部加载

```
启动应用
    ↓
加载 App.vue (3666行)
    ├─ 加载所有知识图谱数据 (2000+行)
    │  ├─ Java并发 (500行)
    │  ├─ 集合类 (300行)
    │  ├─ JVM (400行)
    │  └─ ... 其他板块
    ↓
渲染界面

问题：
❌ 初始加载慢
❌ 内存占用大
❌ 用户可能只看一个知识点
```

### 重构后：按需加载

```
启动应用
    ↓
加载配置文件 (极小)
    ├─ 知识清单
    └─ 知识图谱配置 (只有元数据)
    ↓
渲染界面 (快速)
    ↓
用户点击知识块
    ↓
动态加载内容文件 (~50行)
    ├─ 显示加载状态
    ├─ 加载 description
    └─ 加载 code
    ↓
渲染内容

优势：
✅ 初始加载快
✅ 内存占用小
✅ 按需加载
✅ 用户体验好
```

## 维护成本对比

### 修改一个知识点的内容

#### 重构前
```
1. 打开 App.vue (3666行)
2. 搜索知识点 ID
3. 在大文件中定位内容
4. 修改内容
5. 保存大文件
6. 等待 IDE 重新索引

耗时：5-10 分钟
风险：可能影响其他内容
```

#### 重构后
```
1. 打开对应的内容文件 (~50行)
2. 直接修改
3. 保存

耗时：1-2 分钟
风险：独立文件，不影响其他内容
```
